# Zenn記事 文体ガイド

過去記事（https://zenn.dev/lclco）から抽出した文体ルール。
記事ドラフト生成時にこのスタイルに合わせること。

## 基本方針
- ベースは「丁寧な技術解説」。読者に正確な情報を届けることが最優先
- その上で、人柄がにじむ程度にカジュアルさを加える
- 技術記事としての信頼感 > 親しみやすさ。バランスは 7:3 くらい

## 一人称
- 「わたし」に統一する
- 「私」「筆者」「自分」は使わない
- 例: 「わたしはこれまで〜を使ってきました」

## 文末表現
- 基調は「です/ます」
- 口語は控えめに混ぜる: 「〜ですね」「〜かなと思います」
- 感情表現は最小限にとどめる。1記事に2〜3箇所まで
- 「爆速」「神機能」「最高」のような強い表現は基本使わない

## 絵文字
- 記事全体で3〜5個程度。見出しには使わない
- 使う場面: まとめの締め、注意喚起、感想を一言添えるとき
- 使わない場面: 技術的な説明の途中、見出し、コードブロック前後

## 導入部の書き方
- 共感 or 自分の体験から入る: 「皆さん、〜していますか？」「わたしは普段〜を使っているんですが」
- 課題提示 → 本題、の流れ。前置きは短く
- 導入の最後に何を書くかを自然な文章で軽く触れる程度でよい

## 見出しの書き方
- 内容が伝わる具体的な見出し: 「始め方（3ステップ）」「Subagentsとの違い」
- 体言止めOK: 「セットアップ手順」
- 疑問形もOK: 「なぜ〜なのか？」
- カジュアルすぎる見出しは避ける（「〜からの卒業」程度はOK）

## コードブロックの前後
- 前: 目的を1行で簡潔に。「以下コマンドでインストールできます」
- 後: 補足があればシンプルに。長い説明は不要

## 読者への呼びかけ
- 押し付けない提案型: 「ぜひ試してみてください」「気になった方はぜひ」
- 最後は感謝で締める: 「ありがとうございます🙏」

## 全体のトーン
- 「先輩エンジニアが丁寧に教える」くらいの距離感
- 技術情報は正確に。主観は「〜と感じています」「〜かなと思います」で区別する
- 注意点やデメリットも正直に書く
- 過度な盛り上げはしない。淡々としすぎず、はしゃぎすぎず

## 外部リンクの貼り方
- URLはテキストリンク `[タイトル](URL)` にしすぎない
- 参考記事や公式ページは、URLを単独行で直貼りする（ZennのUIでカード表示になる）
- テキストリンクを使うのは、文中で自然に参照する場合のみ
- 例:
  - カード表示にしたいとき → URLだけの行を置く
    ```
    https://www.anthropic.com/engineering/building-c-compiler
    ```
  - 文中で軽く参照するとき → テキストリンク
    ```
    [公式ドキュメント](https://code.claude.com/docs/en/agent-teams)によると〜
    ```

## NG
- 硬い論文調（「〜である」「〜と考えられる」）
- 「筆者」「本記事」のような硬い表現
- 上から目線（「〜すべきだ」「〜しなければならない」）
- ポップすぎる表現の連発（「🔥」「やばい」「神」を連打しない）
- 絵文字の多用（見出し・本文中に散りばめない）
- AI生成感のある定型パターン（「この記事でわかること:」「本記事では〜を解説します」「〜についてまとめました」等）
