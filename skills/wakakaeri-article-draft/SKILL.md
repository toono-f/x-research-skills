# Wakakaeri Article Draft（note記事作成）

## Overview

x-deep-research の結果を元に、noteに投稿できるレベルの若返り・アンチエイジング記事ドラフトを生成する。
Markdown で出力し、noteにそのまま貼り付けて使える状態にする。

## When To Use

- x-deep-research で若返り・アンチエイジングテーマを深掘りした結果が手元にあるとき
- wakakaeri-post-draft のパターンCで概要を作成した後、本格的な記事にしたいとき
- X投稿で反響があったテーマを note 記事に昇格させたいとき

## Intake

必須:
- deep-research の結果ファイル（`data/deep-research/*_research.md` の最新、またはユーザーが指定）

確認（順番に聞く）:
1. 「どのリサーチ結果を使う？」（ファイル指定 or 最新を自動選択）
2. 「このテーマで自分が試したこと・感じたことは？」（実体験があれば記事に織り込む）
3. 「記事の切り口は？」（以下から選択 or 自由指定）
   - 最新研究まとめ（研究結果を分かりやすく解説）
   - 実践レポート（自分で試した結果を共有）
   - 比較・検証（A vs B を調べた結果）
   - 初心者ガイド（始め方を丁寧に解説）

## Inputs

1. Deep Research レポート: `data/deep-research/*_research.md`
2. 文体ガイド: `skills/wakakaeri-article-draft/references/writing-style-note.md`（**必ず参照**）
3. （任意）ユーザーからの追加コンテキスト（実体験、感想）
4. （任意）投稿案: `data/wakakaeri-post/*_若返り投稿案.md`（あれば切り口の参考にする）

## Workflow

1. **リサーチ結果の読み込み**
   - 指定された deep-research レポートを読む
   - テーマ、エビデンス、リスク、実践ポイントを把握する

2. **文体ガイド参照**
   - `skills/wakakaeri-article-draft/references/writing-style-note.md` を読み、文体ルールを確認する
   - 記事全体をこの文体ガイドに合わせて書く

3. **切り口の決定**
   - ユーザーに切り口を確認する
   - 実体験があればそれを軸にする

4. **記事構成の設計**
   以下の構造で記事を組み立てる:

   **導入（200〜400字）**
   - フック: 読者の関心を引く1〜2文
   - 背景: なぜ今このテーマか
   - この記事で分かることを自然に触れる

   **本論（2000〜4000字）**
   - 見出しは4〜6個
   - 各セクションにエビデンス（出典付き）を含める
   - トレードオフ（メリット/デメリット、リスク）を必ず含める
   - 実践のポイント（読者がすぐ取り入れられるアクション）を含める
   - 専門用語は初出時に平易な言い換えを添える

   **まとめ（200〜400字）**
   - 要点を3行以内で
   - 実践のファーストステップを提案する
   - 医療免責を自然に添える

5. **文体チェック**
   - `writing-style-note.md` に照らして以下を確認:
     - 一人称が「わたし」に統一されているか
     - 文末が「です/ます」基調 + 口語混じりになっているか
     - 導入が共感 or 体験から入っているか
     - 絵文字が5-8個程度入っているか（見出しにも可）
     - 専門用語に平易な言い換えが添えられているか
     - 医療免責が含まれているか
   - 違反があれば修正してから出力する

6. **NGチェック**
   - 医療免責に反する断定表現がないか
   - 出典のない数字・仕様がないか
   - 煽り表現がないか

## Output

`data/article-draft/` に保存:
- ファイル名: `YYYYMMDD_HHMMSSZ_{topic_slug}_article.md`

出力形式（note用 Markdown）:

```markdown
# 記事タイトル

（導入: フック + 背景 + この記事でわかること）

## 見出し1

（本論）

## 見出し2

（本論）

## 見出し3

（本論）

## まとめ

（要点 + 実践のファーストステップ + 医療免責）

## 参考

- [タイトル](URL)
- ...

---

> この記事は一般的な情報提供を目的としたものであり、医学的な診断・治療の助言ではありません。体調に不安がある方は、かかりつけの医師や専門家にご相談ください。
```

### 補足出力

記事本体の後に、以下をコメントブロックで付記する:

```markdown
<!-- draft-meta
## タイトル候補（3案）
1. ...
2. ...
3. ...

## 文体チェック結果
- [ ] 一人称「わたし」統一（私/筆者/自分はNG）
- [ ] です/ます基調（である調なし）
- [ ] 絵文字は記事全体で5〜8個
- [ ] 見出しに絵文字が入っている
- [ ] 専門用語に平易な言い換え添付
- [ ] 医療免責が記事末尾にある
- [ ] 煽り表現なし（驚愕/衝撃/〜するだけ等）
- [ ] 硬い表現なし（本記事/筆者/〜すべき等）

## NGチェック結果
- [ ] 断定的な効果表現なし
- [ ] 煽り表現なし
- [ ] 出典なしの数字なし
- [ ] 副作用・リスクへの言及あり

## 元データ
- Based on: {deep-research ファイルパス}
- Style guide: skills/wakakaeri-article-draft/references/writing-style-note.md
-->
```

## 記事の品質基準

- 文字数: 3000〜5000字
- 見出し: 4〜6個（スクロールで構造がわかる）
- 出典: エビデンスには必ずURLまたは出典情報を付記
- 文体: `writing-style-note.md` に準拠
- エビデンスレベル: 臨床試験 / 動物実験 / 観察研究 / 専門家見解 を明記
- リスク・注意点: 効果だけでなくデメリットを必ず書く
- 医療免責: 記事末尾に免責事項を記載
- 実践性: 読者がすぐに取り入れられるアクションを含める

## Hand-off

- 投稿: ユーザーが仕上げて note に投稿（人間作業）
- X告知: 公開後に wakakaeri-post-draft で告知ポストを作成
