# X Post Draft（投稿作成）

## Overview

x-deep-research の結果と発信者の実体験を組み合わせて、「自分にしか書けない投稿」の下書きを作る。
一般論ではなく、実体験・逆張り・実装Tipsの3切り口で投稿案を生成する。

## When To Use

- x-deep-research の結果が手元にあるとき
- テーマは決まっているが、どう投稿するか迷っているとき

## Intake

必須:
- deep-research の結果ファイル（`data/deep-research/*_research.md` の最新、またはユーザーが指定）

確認:
- 「どのリサーチ結果を使う？」（ファイル指定 or 最新を自動選択）
- 追加の実体験があれば聞く（「このテーマで自分がやったこと・感じたことは？」）

## Inputs

1. Deep Research レポート: `data/deep-research/*_research.md`
2. 発信者プロフィール: `skills/x-post-draft/references/profile.md`
3. （任意）ユーザーからの追加コンテキスト（実体験、感想）

## Workflow

1. **リサーチ結果の読み込み**
   - 指定された deep-research レポートを読む
   - テーマ、ファクト、賛否、空白地帯を把握する

2. **プロフィール参照**
   - `references/profile.md` を読み、発信者の立場・トンマナ・NGを確認する

3. **追加コンテキスト確認**
   - 「このテーマで何か実体験や感想はある？」とユーザーに聞く
   - あれば切り口に反映、なければリサーチ結果の空白地帯から切り口を構成

4. **3パターンの投稿案を生成**

### 切り口パターン

**パターン A: 実体験ベース**
- 「実際にやってみたら〜だった」
- 自分のスタック（TypeScript/Next.js、AI/LLM）と絡める
- 過程を見せるスタイル

**パターン B: 逆張り / 意外性**
- 「みんな〜と言うが、実際は〜」
- リサーチの反論/批判をベースに、自分なりの視点を加える
- 賛否が分かれるポイントで独自ポジションを取る

**パターン C: 実装Tips / ハウツー**
- 「〜するなら、この構成が最小」
- 具体的なコード・設定・手順を含む
- 再現性が高い情報

### 各パターンの構成

- フック文（1行目）を3案ずつ（計9案）
- 本文案（140〜280文字目安、X投稿1ポスト分）
- 補足: スレッド展開する場合の2ポスト目案（任意）

5. **NGチェック**
   - profile.md の NG 事項に抵触していないか確認
   - 断定（「〜は確実に〜」）→ 柔らかい表現に修正
   - 煽り（「知らないとヤバい」）→ 事実ベースに修正
   - 投資助言 → 削除

## Hand-off

- 投稿: ユーザーが自分の言葉で仕上げてXに投稿（人間作業）
- 記事化: 反響が大きければ `article-agent-context-research` → `article-agent-outliner` へ
