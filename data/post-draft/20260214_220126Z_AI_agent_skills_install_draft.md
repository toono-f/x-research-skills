## パターン A: 実体験ベース

### フック文候補
1. Claude Codeを「AIエージェント」として運用し始めてから、開発の動き方が根本的に変わった。スキルの追加がプラグイン感覚で回る
2. Claude CodeにAgent Skillsを足していくと、できることが段階的に広がっていく。最初はコード生成だけだったのが、外部ツール連携まで自然に伸びた
3. AIエージェントスキルのインストール、構造さえ分かれば「APIキー設定 → スキル登録」でほぼ終わる。試すまでが一番遠い

### 本文案
Claude CodeにAgent Skillsを追加して運用してみている。

やっていることはシンプルで、Claude Agent SDK経由でスキル（connectors）を登録して、ツールコールで外部サービスと連携させる流れ。ヘルスケア分野では12種以上のconnectorsが公式提供されていて、API経由でインストールできる。

実際に触ってみて感じたのは、Claude Codeの連続ツールコール（tool calls）がかなり安定しているということ。Anthropicの内部データ（20万セッション分析）でも、人間の介入ターンが減ってエージェント的な動きが強まっている傾向が出ている。

一方で、日本語圏の実装例はほぼ見つからない。公式ドキュメントもニュースリリース中心で、ハンズオン的な情報は薄め。ここは自分で試して埋めていく余地がありそう。

Anthropic内部の132名エンジニア調査では、Claude Codeで「専門外のタスクも処理できるようになった」という声が多い。研究員がフロントエンドを作ったり、非技術者がGitデバッグしたり。

ただし53名のインタビューでは「ハンズオンの満足感が下がった」「スキルの鋭敏化が鈍る」という声も出ている。レビュー工程を意識的に挟むのが大事らしい。

## パターン B: 逆張り / 意外性

### フック文候補
1. AIエージェントスキルの話題で「何ができるか」ばかり注目されるけど、実際に差がつくのは「どうインストールして保守するか」の設計だと思う
2. Claude Codeを使いこなしてる人ほど、スキル追加に慎重な印象がある。入れすぎるとtool callsが不安定になるリスクもある
3. 「AIエージェントで自動化」と聞くと華やかだけど、Anthropic内部調査では「スキル鈍化」を感じているエンジニアが結構いる

### 本文案
AIエージェントスキルの話、「何ができるか」に目が行きがちだけど、実務で効くのは「インストール設計」のほう。

Claude CodeのAgent Skillsはconnectors形式でプラグイン的に足せる。ただ、入れれば入れるほど良いわけじゃない。連続tool callsの安定性はスキル数やコンテキストの複雑さに左右される。

Anthropicが社内132名のエンジニアを調査した結果が興味深くて、Claude Codeで専門外タスクの処理が可能になった反面、「自分で手を動かす感覚が鈍る」という53名分のインタビューデータが出ている。

つまり、スキルを足して自動化を進めるほど、人間側の「レビュー力」が重要になる。エージェントに任せきりにしない設計が、実は一番の実務インストール術なのかもしれない。

### スレッド展開案（任意）
代替手段との比較もしておくと参考になるかも。

- Claude Code (Agent SDK): IDE統合が強み（Xcode対応済み）。connectors形式でスキル登録
- OpenAI Assistants API: Tools JSONで標準的。VS Code拡張あり。ドキュメント充実
- LangChain Agents: 自由度は最も高い。OSS。ただしセットアップは手動寄り

選び方としては、Apple開発ならClaude SDK、ドキュメント重視ならOpenAI、カスタマイズ優先ならLangChain、という使い分けになりそう。

## パターン C: 実装Tips

### フック文候補
1. Claude CodeでAgent Skillsを試す最小構成。APIキー取得 → SDK確認 → スキル登録テスト、の3ステップ
2. AIエージェントのスキル追加、まだ日本語の実装情報がほぼない。最小手順をまとめておく
3. Claude Agent SDKでスキルをインストールするなら、この順番で進めるのが手戻り少ない

### 本文案
Claude CodeでAgent Skillsを試す最小手順をまとめておく。

1. APIキー取得: Anthropic Console (claude.ai) で発行
2. SDK確認: Claude Agent SDKのドキュメントをチェック。Xcode統合済みならプラグインインストールだけで動く
3. スキル登録テスト: connector定義（JSON）を用意してAPIエンドポイントにアップロード → tool callで動作確認

現状の注意点として、
- 公式ドキュメントはニュースリリース中心。ハンズオン情報は少なめ
- Xcode統合は確認済みだけど、VS Code等の他IDEは公式未言及
- レートリミットや料金の詳細は未公表

日本語圏の実装例はほぼゼロなので、試した知見を残しておくだけで価値がある領域。まだ誰も検証していないBun.js買収後のJS/TSパフォーマンスや、Vision Pro向けAgent Skillsあたりは差別化しやすそう。

### スレッド展開案（任意）
未踏の検証テーマとして面白そうなもの。

- Bun.js買収後のClaude CodeのJS/TSパフォーマンス。Node互換性やサーバーレスエージェントの高速化がどう変わったか
- Apple Vision Pro専用のAgent Skills。空間コンピューティング連携でAR/VRエージェントの新しいUXが生まれる可能性
- 日本語圏ヘルスケアconnectorsのローカライズ。12+スキルの多言語対応は未確認で、日本の医療DBとの連携は差別化要素になりそう
